### 교육 내용

#### 교육목적:
- 단위 테스트의 중요성과 목적을 이해한다.
- 효율적이고 유지보수 가능한 단위 테스트 코드 작성 방법을 학습한다.
- TDD 방법론을 실무 프로젝트에 적용할 수 있다.
- 테스트 자동화 및 CI/CD 환경 구축 실습을 통해 품질 중심의 개발 프로세스를 체득한다.

#### 교육구성:
- 단위 테스트 기본 원리와 C# 기반 테스트 프레임워크 (NUnit/xUnit) 사용법을 학습합니다. TDD의 Red - Green - Refactor 사이클을 실습합니다.
- 테스트 주도 개발의 핵심 개념을 익힙힙니다. Mocking, Stubbing, Fake 객체 활용을 통해 외부 의존성을 격리하고, 복잡한 로직 (API, 데이터 연동 등)을 효율적으로 테스트하는 전략을 학습합니다.
- 코드 커버리지 도구 (ReportGenerator)**를 활용해 테스트 품질을 측정/분석하고, 리포트 결과를 기반으로 테스트를 개선하는 방법을 실습합니다.
- Github Actions / Jenkins 를 이용해 테스 자동화 및 CI/DC 파이프 라인을 구축하고, 테스트 실행 -> 결과 리포팅 -> 품질 피드백까지의 전 과정을 체험합니다.
- 팀별 미니 프로젝트를 통해 실무형 TDD 적용 및 코드 리뷰를 수행하며, 테스트 자동화와 협업 중심 개선 역량을 강화합니다.

#### 교육목표:
- 단위 테스트 및 TDD 기반 코드 작성 능력을 습득할 수 있다
- 외부 의존성 격리와 복잡한 비즈니스 로직 테스트 전략을 이해할 수 있다
- 테스트 자동화 , 코드 커버리지 분석 및 실무 적용 능력을 향상할 수 있다

- 단위테스트의 필요성 및 목적 , TDD 개념과 절차
- FIRST 원칙 (Fast, Independent, Repeatable, Self Validating, Timely)
- NUnit / xUnit 구조 이해

#### 2일차: TDD 실전 적용 및 코드 커버리지
- Red Green Refactor 단계별 실습 
- 실패하는 테스트 작성부터 리팩토링까지

#### 실습/교안 예시
- FIRST 원칙 단위테스트에서의 기본 원칙
- Fast (빠르게 ): 단위 테스트는 빠르게 실행되어야 합니다. 그래야 개발자가 즉각적으로 피드백을 받을 수 있고 TDD 와 같은 개발 방법론을 효과적으로 적용할 수 있습니다 .
- Independent (독립적으로 ): 각 테스트는 다른 테스트의 결과에 영향을 받지 않아야 합니다. 즉, 테스트 실행 순서에 상관없이 항상 동일한 결과를 보장해야 합니다 .
- Repeatable (반복 가능하게 ): 테스트는 어떤 환경에서든 동일한 결과를 내야 합니다.
- Self Validating ( 자가 검증 ): 테스트는 성공 또는 실패를 명확하게 알려줘야 합니다.
- Timely (적시에 ): 테스트 코드는 테스트 대상 코드가 작성되기 직전에 만들어져야 합니다. 이는 TDD 테스트 주도 개발 의 기본 원칙이기도 합니다 .
